# 擴散 Rubric（讓圖卡被存、被傳、被買）

> 這份管「社群機制與擴散」，跟 `style-system.md`（管視覺）、`content-planning.md`（管拆頁）並列。
> 概念改寫自 [DennisWei9898/social-cards-engine](https://github.com/DennisWei9898/social-cards-engine)（MIT）的知識輪播 rubric，在地化為台灣 IG／Threads／Facebook。

本技能原本強在**設計**，弱在**擴散**。這份 rubric 補上擴散層：把「會被演算法推、會被朋友轉」的機制，接到 Editorial / Swiss 兩套視覺系統上。**設計歸設計，擴散歸擴散，兩者疊加。**

## 🔩 演算法訊號優先序（設計時的北極星）

IG 官方（Mosseri）定性排序，2026：

**私訊分享 sends ＞ 儲存 saves ＞ 5 字以上留言 ＞ 公開分享 ＞ 一般留言 ＞ 讚≈0**

知識型輪播的設計目標，照這個順序：
1. **讓人「傳給某個特定的人」**（sends，權重最高）——封面與收尾要有「提醒誰」的轉傳鉤。
2. **讓人「存起來」**（saves）——要有工具性內容 + 速查總表 + 收藏指令。
3. 引導留言關鍵字。
讚不是目標，不要為按讚設計。

## 🧱 擴散結構模板（知識型 8–10 張）

不取代 `layout-recipes.md` 的版型，而是替每頁**分配一個擴散角色**（版型照舊挑）：

| 位置 | 角色 | 要點（接哪個 recipe） |
|---|---|---|
| 1 封面 | 鉤子＋署名＋收藏訊號 | 好奇缺口／損失規避標題（**非類目名**）＋品牌名＋「📌建議收藏」。Editorial 用 M01/M16；Swiss 用 S01 |
| 2 | 認知地基 | 「這是什麼／為什麼該在意」，比喻先行。M03/M11 或 S02 |
| 3–N | 工具卡群 | 份量／挑選／保存／對比——「之後會回來查」＝save 引擎。M05/M08 或 S10/S11 |
| 中段 | 照片喘息卡 | 每 2–3 張文字後插一張真實照片卡打斷節奏。M02 |
| 倒數第 2 | **速查總表卡** | 分散資訊併成「一張截圖帶走」，標「存這張」。M05/M08 或 S12 |
| 分享卡 / 收尾 | 點名對象 + 品牌 CTA | 「把這張傳給那個…的朋友 →」觸發最高權重的私訊分享。M04/M07 或 S07 |

> Editorial 的克制美學仍在——**擴散元件用該系統的語彙做**：收藏指令用 `.label`（mono），不要貼一顆突兀的 emoji 泡泡到雜誌頁上；Swiss 的 CTA 可以是一個 `.card-accent` 區塊。別為了擴散破壞視覺系統。

## ✅ 發布前 10 條硬 checklist（逐條過）

1. **封面是「鉤子」不是「類目」**：命中好奇／FOMO／損失規避之一；主結論字級最大最先讀到；避免「XX 完整解析」這種目錄式。
2. **封面即帶品牌署名**：單張被轉傳時仍認得出是誰（品牌名／wordmark，非浮水印級）。
3. **封面明示「可存」＋收藏指令**：saves 不要只在結尾才喊。
4. **每張一個概念**：主標一句、內文盡量 ≤30 字、主數字放大、次要資訊降級為小字。
5. **有節奏起伏**：照片／文字／數據交錯，中段不要連 4 張以上同版型。
6. **主副層級對比夠**：手機縮到約 20% 仍讀得到主結論（Editorial 淺底忌太淺灰，往深靠）。
7. **至少一張速查／總表卡**：讓人一張截圖帶走、利於單張轉傳。
8. **結尾 CTA 一主一輔、動詞前置、附收益理由**：別三條並列。
9. **有一張給「該提醒誰」的轉傳指令**：點名對象（長輩／新手／同事／最近很累的朋友）拉私訊分享。
10. **知識與商品有橋樑**：中段用知識「反證」產品選擇，別到最後才硬接導購。

## ⚠️ 台灣 IG 硬規則（機器可判，交給 `validate-social-deck.mjs` 與 reviewer 硬項）

- **尺寸**：知識型 4:5（1080×1350）滿版；快速分享可 1:1。
- **輪播不被裁鐵律**：整套「每張同一比例」就不會被切（IG 以第一張比例套用全部）。**全 4:5 或全 1:1，別混用。**
- **Hashtag ≤5 個**（精準分類，非塞熱門；超過易被判 spam）。
- **字級**：body ≥24px、headline ≥36px（本技能 Editorial/Swiss 預設字級遠大於此，天然達標）。
- **正文不放可點連結**：導 bio／限動貼紙／留言。

## 🚫 誠實紅線（別為擴散造假）

- **刪除杜撰數字**：「80% 的人」「掉 40% 完讀」「第一小時決定一切」都是迷思，只當方向、**不寫死、不當承諾**。
- 無 engagement bait（「tag 50 人抽獎」「10 秒內標記」）。
- 功效／效果類（水晶、保健、美妝）一律寫**傳統寓意／使用心得**，明白標注**非療效**。
- 內容原創，非搬運。

## 一句話

知識型圖卡的贏法＝**封面下對鉤子、中段給工具、倒數給總表、收尾給轉傳指令、全程掛品牌**，對著演算法最貴的「私訊分享 + 收藏」去設計——但**用你選的視覺系統的語彙去做，不破壞設計**。
