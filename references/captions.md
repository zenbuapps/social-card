# 4 平台文案（IG / Threads / Facebook / X）

> 圖卡出完後，一併產出 `social_captions.md`：每個平台一個 code block，方便直接複製貼上。
> 概念改寫自 [DennisWei9898/social-cards-engine](https://github.com/DennisWei9898/social-cards-engine)（MIT）。搭配 `spread-rubric.md` 的訊號優先序（sends＞saves＞likes）。

## 通則
- **首句呼應封面鉤子**——文案開頭不要重講標題，要接著鉤子往下勾。
- **一主一輔 CTA，動詞前置，附收益理由**（「傳給那個…的朋友」＝sends；「先收藏，之後對照」＝saves）。
- **正文不放可點連結**（一般帳號正文連結不可點）→ 導 bio／限動貼紙／留言關鍵字。
- **不寫杜撰數字**（見 spread-rubric 誠實紅線）；功效類標「非療效／使用心得」。
- 台灣用語、口語親切像朋友提醒，不要教科書腔。

## 各平台配方

### Instagram（貼文）
150–300 字。結構：鉤子首句 → 3–5 行重點（可用 emoji 分行）→ 速查提示（「滑到第 N 張有…截圖存起來」）→ sends CTA →（換行）導購導 bio → hashtag ≤5 集中末尾。

```
（鉤子首句，呼應封面）

（3–5 行重點，一行一觀念）

滑到第 N 張有一張速查總表，先截圖存起來 📌

把這篇傳給那個（點名對象）的朋友 ✨
—
👉（品牌／導購）：@handle（連結在 bio）

#標籤1 #標籤2 #標籤3 #標籤4 #標籤5
```

### Threads
短、口語、可討論。開頭一句鉤子 + 直接開清單 + 收尾問句（催 5 字以上留言）。**Hashtag 0–1 個**（Threads 不吃 hashtag 堆）。

```
（一句反常識鉤子）👇

（3–4 行條列，最精簡）

（一句收尾問句，例：你最想要哪一個？）
```

### Facebook
可長、可帶連結（FB 正文連結可點，但會壓觸及——導購連結放留言第一則更好）。結構：粗體標題行 → 段落展開 → 條列 → 收尾 CTA。語氣比 IG 再完整一點，適合年齡層偏長。

```
【（標題行）】

（一段引子，點出痛點／情境）

・（重點 1）
・（重點 2）
・（重點 3）

（保養／注意事項一段，如適用）

覺得有用的話，傳給（點名對象）吧 🙂
（品牌一句）→ 連結放留言
```

### X（Twitter）
最短。一句鉤子 + 極簡清單，280 字內，最多 1–2 個 hashtag。

```
（一句鉤子）：

（4 行極簡清單）

（收尾一句：存起來／傳給誰）
```

## 產出後檢查（交給 reviewer 的 caption 項）
- IG 首句有鉤子、150–300 字、結尾明確 CTA（E3）。
- Hashtag：IG ≤5、Threads 0–1、X ≤2（C3）。
- 有 sends CTA（提醒誰）＋ saves CTA（先收藏）（D1/D2）。
- 無杜撰數字、無 engagement bait（C2/C5）。
- 正文無可點連結、導 bio／留言（D3）。
