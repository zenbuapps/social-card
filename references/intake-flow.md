# 需求釐清對話流程（Intake Dialogue）

> 動工前的對話腳本。**目的分流是硬閘**：沒弄清楚目的之前，不寫任何 HTML、不算任何圖。
> 其餘問題「缺啥問啥」——使用者訊息裡已經講清楚的，不再重問。

## 為什麼要這個流程

「幫我做一張圖」有太多種意思：一張純粹漂亮的品牌畫面、一組要被轉傳的知識輪播、一張要賣東西的導購卡——三者的結構、文案、CTA、審稿深度完全不同。先分流，才不會做出「漂亮但不對題」的東西。

## 提問方式

- 用**選項題**問（每個問題給 2–4 個選項），不要開放式地問「你想要什麼」。
- **一輪最多 2–4 題**，按輪次進行，不逐題轟炸也不一次全灌。
- **跳過已知**：使用者訊息裡已明說的資訊（目的、平台、風格、素材），直接記下、不再問。
- 「都可以」「你決定」= 接受推薦選項繼續，**之後不再重問同一題**。
- 每題若有明確建議，把推薦選項放第一個並標「（推薦）」。

## 第 0 輪 · 目的分流（必問硬閘）

> 「這次要做的是哪一種？」

| 選項 | 一句話 | 解鎖的流程 |
|---|---|---|
| **A 形象美圖** | 只要漂亮、有質感的品牌畫面，**不需要 CTA** | 只問視覺（第 1、2 輪），**跳過擴散層**；出圖後只跑版面 validator |
| **B 擴散型內容** | 要被存、被轉傳的知識／攻略輪播 | 完整流程：`spread-rubric.md` 結構（鉤子＋速查總表＋轉傳 CTA）＋ 自動過 `social-card-reviewer` ＋ 4 平台文案 |
| **C 導購／活動** | 賣商品或宣傳活動，要明確的行動 | 同 B，另加問：檔期／價格／連結／主打商品；CTA 主力用導購型（購買／報名／限時） |
| **D 先看提案** | 還不確定方向 | **不先算圖**：用文字描述 2 種方向的提案（各自的風格、結構、頁數、範例標題），使用者選定後回到 A/B/C 對應流程 |

例外：使用者訊息已明說目的（例：「做一套會擴散的輪播」→ B；「就做一張美美的形象圖」→ A），直接分流、不問。

## 第 1 輪 · 視覺基礎（2 題一起問；已知則略）

**版位（可多選）**
- 4:5 直式貼文／輪播（1080×1350，主力推薦）
- 1:1 方形貼文（1080×1080）
- 9:16 限時動態／Reels 封面（1080×1920）
- 貼文＋限動組合（4:5 ＋ 9:16 成對）

**風格**
- Editorial 雜誌風（襯線、紙墨、克制敘事；6 配色見 `theme-presets.md`）
- Swiss 國際主義（網格、單一錨點色、數據感；4 錨點色）
- 不確定 → 依內容型別推薦一種並說明理由（不用先算圖）

## 第 2 輪 · 素材（一次問完；已知則略）

沿用 SKILL.md 的「三選一」規則（只問一次，不重複推銷）：
- **A** 自己有照片／截圖（推薦——最不「AI 感」）
- **B** 幫我從圖庫找（Pexels／Unsplash／Flickr CC）
- **C** AI 生成
- **D** 純排版不用圖（Swiss 數據卡、文字卡適用）

## 第 3 輪 ·（條件式）擴散配置 — 只有目的 = B 或 C 才問

- **CTA 主力**（一主一輔，見 `spread-rubric.md`）：
  - 轉傳型「把這張傳給某人」（餵 sends，權重最高）
  - 收藏型「先存起來之後用」（餵 saves）
  - 導購型「連結在 bio」（C 預設主力）
  - 留言型「留言告訴我…」
- **要不要 4 平台文案**（IG／Threads／FB／X，見 `captions.md`）——B/C 預設要。
- 審稿員**不問**：B/C 出圖後自動過 `social-card-reviewer`（寫審分離，最多 3 輪）；A 只跑 `validate-social-deck.mjs` 版面把關。

## 收尾 · 計畫確認（動工前最後一閘）

問完後，**先給一張「頁面計畫表」再算圖**：

```text
Page 01 / 角色（鉤子封面）/ 版型 M16 / 標題草案 / 圖片來源
Page 02 / 角色（認知地基）/ 版型 M03 / …
...
Page 08 / 角色（轉傳 CTA）/ 版型 M04 / …
```

使用者點頭（或說「可以」「開始吧」）才進入算圖。若使用者對計畫有意見，改計畫、不重問已答過的題。

## 流程總覽

```
使用者觸發 skill
   │
   ▼ 第 0 輪：目的分流（必問，除非已明說）
   A 形象美圖 ──► 第 1、2 輪（視覺＋素材）──► 計畫確認 ──► 出圖 ──► validator
   B 擴散內容 ──► 第 1、2 輪 ──► 第 3 輪（CTA＋文案）──► 計畫確認 ──► 出圖 ──► validator ──► 審稿員（自動）──► 文案
   C 導購活動 ──► 同 B，另補檔期／價格／連結
   D 先看提案 ──► 文字提案 2 方向 ──► 使用者選定 ──► 回到 A/B/C
```
